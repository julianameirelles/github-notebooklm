Mini Guia:
Fala, galera da tecnologia! Sejam bem-vindos a mais um vídeo aqui no canal! Hoje eu vou explicar para vocês o que é esse tal de GitHub e como você pode sair do zero e começar a usar essa ferramenta incrível em apenas 10 passos. 🚀

Para começar, o GitHub é como uma "garagem" onde você guarda o seu carro, sendo que o motor desse carro é o Git
Ele é uma plataforma online usada para hospedar códigos e arquivos, funcionando como uma verdadeira "rede social para desenvolvedores", onde você pode colaborar em projetos do mundo inteiro

Bora para o mão na massa? Segue esse roteiro de 10 passos:

1) Crie sua conta: Acesse o site oficial (github.com) e faça seu cadastro gratuito. É rápido, fácil e será sua identidade como dev

2) Instale as ferramentas: Você precisa do Git instalado no seu computador
Se você não curte usar linhas de comando no terminal, pode baixar o GitHub Desktop, que tem uma interface visual muito mais amigável

3) Configure seu perfil: Abra o terminal e "se apresente" para o Git configurando seu nome e e-mail global. Isso é importante para que o sistema saiba quem fez cada alteração no código

4) Crie um Repositório: No site do GitHub, clique em "New" para criar uma pasta para o seu projeto (o repositório). Dê um nome, uma descrição e marque a opção de adicionar um arquivo README, que serve como a "capa" do seu projeto

5) Traga o projeto para o seu PC (Clone): Clique no botão verde "Code" no site, copie o link e use o comando git clone no terminal para baixar uma cópia completa do repositório para o seu computador

6) Mão no código: Agora é hora de trabalhar! Abra a pasta no seu editor favorito (como o VS Code) e crie ou edite seus arquivos

7) Prepare as alterações (Add): O Git não salva nada sozinho. Use o comando git add para colocar os arquivos que você mexeu na "área de preparação" (staging area), avisando que eles estão prontos para o próximo passo

8) Grave o histórico (Commit): Use o comando git commit -m "sua mensagem". O commit funciona como um "snapshot" (uma foto) do seu projeto naquele momento, registrando o que foi feito

9) Suba para a nuvem (Push): Suas alterações ainda estão só no seu PC. Use o comando git push para "empurrar" seus commits para o repositório remoto no GitHub para que o mundo todo (ou sua equipe) possa ver

10) Mantenha-se atualizado (Pull): Se outra pessoa mexeu no código, você precisa baixar essas novidades. Use o comando git pull para sincronizar seu computador com a versão mais recente que está no GitHub

E pronto! Com esses 10 passos você já está dominando o fluxo básico do GitHub. Não esquece de deixar o like, se inscrever no canal e compartilhar com aquele amigo que está começando a programar! Valeu e até a próxima! ✌️💻
